# 정보구조 · 사용자 플로우 — 맑은스튜디오 (IA & UX)

> 목적: 4대 해자(그라운딩·스킬 개인화·표준 발행·HITL 검수)를 실제 사용자가 밟는 화면·플로우로 번역한다.
> 정본 참조: 01-COMPETITION(경쟁 정본) · 02-STRATEGY(전략). 모든 추정은 "(가정)" 표기.
> 형제 앱: `malgn-studio`(스튜디오 — 제작/검수) · `malgn-studio-admin`(운영자 콘솔) · `malgn-studio-lms`(학습자 LMS).
> (가정) 작성 시점 형제 레포는 미구현 상태 → 본 IA는 그린필드 설계이며, 구현 시 실제 코드 구조와 정합성 재검증 필요.

---

## 1. 목적

- 전략 문서가 정의한 **단일 소스 → 8종 산출물 → 출처 확인 → 검수 → 발행 → 배포 → 수강**의 엔드투엔드 흐름을, 역할·화면·상태 단위로 구체화한다.
- 핵심 차별 UX(소스 그라운딩 + 출처 패널 + HITL 게이트)를 **생성 워크스페이스 한 화면 안**에서 끊김 없이 경험하도록 IA를 설계한다.
- MVP에서 먼저 만들 화면의 우선순위를 정해 개발 범위를 좁힌다(02-STRATEGY §6 속도 전략 정렬).

## 2. 사용자 · 역할

전략 §3의 역할을 화면 권한 단위로 구체화한다. (가정) 역할은 조직 단위로 부여되며 한 사용자가 복수 역할을 겸할 수 있다.

| 역할 | 1차 목표 | 주요 권한 | 주 사용 앱 |
|---|---|---|---|
| 콘텐츠 제작자 / 교수설계자(ID) | 소스로부터 8종 산출물을 생성·편집해 검수 요청까지 올린다 | 프로젝트 생성·소스 관리·생성 실행·산출물 편집·검수 제출·스킬 태깅(초안) | studio |
| 검수자(HITL / SME·리뷰어) | 출처·정확성·교수 적합성을 확인하고 승인/반려한다 | 검수 큐 조회·코멘트·승인/반려·버전 비교·출처 검증 | studio |
| 운영자 / 관리자 | 발행·배포·과금·권한·스킬 택소노미를 운영한다 | 회원·권한·과금·사용량·스킬 택소노미·발행 정책·LMS 배포 | admin |
| 학습자 | 배포된 콘텐츠를 개인화된 형태로 수강한다 | 카탈로그 열람·수강·진도·평가 응시 | lms |
| (보조) 의사결정자/구매자 | 도입·성과를 확인한다 | 사용량·성과 리포트 열람(읽기) | admin (가정) |

권한 모델(가정): `제작자 ⊂ 검수자(겸직 가능) ⊂ 운영자` 의 누적이 아니라 **기능 축별 분리**. 검수자는 자신이 제작한 산출물을 자가 검수할 수 없음(4-eyes 원칙, HITL 신뢰성 보장).

## 3. 핵심 시나리오 (엔드투엔드)

하나의 소스에서 LMS 수강까지. 각 단계의 **화면 · 사용자 행동 · 산출물 상태**를 함께 표기한다.

| # | 단계 | 앱 / 화면 | 사용자 행동 | 산출물 상태 |
|---|---|---|---|---|
| 1 | 프로젝트 생성 | studio / 프로젝트 생성 모달 | 제목·목표 학습자·기본 스킬 입력 | `프로젝트:초안` |
| 2 | 소스 업로드/연결 | studio / 소스 패널 | 문서·PPT·텍스트 업로드, 웹·영상 URL 연결 | `소스:수집중→인덱싱완료` |
| 3 | AI 생성(산출물 선택) | studio / 생성 워크스페이스 | 8종 중 선택(MVP: 요약·슬라이드·퀴즈) + 옵션(난이도·길이) 후 실행 | `산출물:생성중→초안` |
| 4 | 출처 확인 | studio / 출처 패널 | 문장별 인용(citation) 클릭→원본 소스 위치 확인, 무출처 문장 플래그 | `산출물:초안(출처검증)` |
| 5 | 검수(HITL) | studio / 검수 화면 | 검수자가 코멘트·수정요청·승인/반려 | `초안→검수중→승인` 또는 `반려` |
| 6 | 스킬 태깅 | studio / 산출물 편집기(스킬 탭) | 산출물·문항에 스킬 택소노미 매핑 | `승인(스킬태깅됨)` |
| 7 | 발행(네이티브/SCORM) | studio / 발행 → admin 정책 | 발행 포맷 선택(LMS 네이티브 / SCORM·xAPI), 패키징 | `승인→발행됨` |
| 8 | LMS 배포 | admin / 콘텐츠·배포 | 코스 편성·대상 그룹·공개 일정 지정 | `발행됨→배포됨` |
| 9 | 학습자 수강 | lms / 수강 플레이어 | 콘텐츠 소비·진도 적립·평가 응시 | `배포됨→수강중→이수` |

상태머신(산출물 핵심): `초안 → 검수중 → (승인 | 반려→초안) → 발행됨 → 배포됨`. 반려 시 코멘트와 함께 제작자에게 회귀(02-STRATEGY 해자 ④).

예외·분기(가정):
- 출처 미달(무출처 문장 비율 임계 초과) → 검수 제출 전 **경고 게이트**, 강제 재생성 권고.
- 스킬 태깅은 MVP에서 "씨앗" 수준(수동 태깅), 자동 태깅·생성형 변주는 Phase 2(02-STRATEGY §10).

## 4. 화면 정보구조 (IA) — 앱별 화면 트리

### 4.1 studio (제작/검수 앱)

```
studio
├─ 로그인 / 조직 선택
├─ 대시보드
│   ├─ 내 프로젝트 요약(상태별 카운트: 초안/검수중/발행됨)
│   ├─ 검수 대기 큐(검수자 역할 시)
│   └─ 최근 활동 · 사용량 위젯
├─ 프로젝트 목록
│   ├─ 필터(상태·소유자·스킬·기간) · 검색
│   └─ [+ 새 프로젝트]
├─ 프로젝트 (상세)  ★핵심
│   ├─ 소스 패널            (업로드/연결·인덱싱 상태·소스 목록)
│   ├─ 생성 워크스페이스      (산출물 선택·옵션·생성 실행·결과)
│   │   └─ 산출물 편집기      (8종 탭: 요약/슬라이드/퀴즈/문제은행/커리큘럼/콘티/오디오/영상)
│   │       ├─ 본문 편집
│   │       ├─ 출처 패널(citation)
│   │       └─ 스킬 태깅 탭
│   ├─ 검수(HITL)           (검수 요청·코멘트·승인/반려·버전 비교)
│   └─ 프로젝트 설정         (목표 학습자·기본 스킬·협업자)
└─ 발행
    ├─ 발행 포맷 선택(LMS 네이티브 / SCORM 1.2·2004 / xAPI)
    ├─ 패키징 미리보기 · 검증
    └─ 발행 이력
```

### 4.2 admin (운영자 콘솔)

```
admin
├─ 로그인
├─ 대시보드(운영 지표: 사용량·발행·과금 요약)
├─ 회원 관리            (계정·조직·초대·상태)
├─ 권한 관리            (역할 정의·역할-사용자 매핑·4-eyes 정책)
├─ 콘텐츠 관리          (발행물 목록·버전·배포 상태·회수)
│   └─ 배포              (코스 편성·대상 그룹·공개 일정 → lms)
├─ 스킬 택소노미 관리    (스킬 트리 CRUD·매핑 규칙·자동태깅 사전(가정))
├─ 과금 관리            (요금제·크레딧·청구·결제 이력)
├─ 사용량 관리          (생성 크레딧 소비·API 벤더별 비용·조직별 한도)
└─ 운영 설정            (영상/오디오 벤더 연동·발행 정책·LRS 연동(가정))
```

### 4.3 lms (학습자 LMS)

```
lms
├─ 로그인
├─ 코스 카탈로그        (검색·스킬 필터·추천(가정))
├─ 코스 상세            (개요·커리큘럼·선수 스킬·등록)
├─ 수강(플레이어)
│   ├─ 콘텐츠 뷰어(영상/오디오/슬라이드/요약)
│   ├─ 진도 트래커(xAPI/네이티브 진도 적립)
│   └─ 출처 보기(가정: 그라운딩 신뢰 UX를 학습자에도 노출)
├─ 평가              (퀴즈·문제은행 응시·채점·피드백)
├─ 내 학습           (수강 이력·진도·이수증·스킬 성취)
└─ 마이페이지
```

## 5. 주요 화면 와이어프레임 (텍스트)

### 5.1 ★ studio — 생성 워크스페이스 (3~4 패널)

핵심 차별 화면. 소스·생성·출처·검수를 한 화면에서 끊김 없이 오가도록 멀티패널로 구성. (가정) 기본 3패널, 검수 모드 진입 시 4패널.

```
┌─ GNB: 맑은스튜디오 | 프로젝트명 ▾ | [생성] [출처] [검수] | 상태뱃지: 초안 | 사용자 ▾ ┐
├──────────────┬───────────────────────────────┬──────────────────────────────┤
│ ① 소스 패널   │ ② 생성/편집 워크스페이스        │ ③ 출처 패널 (citation)        │
│ (좌, 접힘가능) │ (중앙, 메인)                   │ (우, 토글)                    │
│              │                               │                              │
│ [+ 소스추가]  │ ┌ 산출물 탭 ─────────────┐     │ 선택 문장의 근거:             │
│ ─ 문서/PPT    │ │ 요약 | 슬라이드 | 퀴즈 |…│     │ ▸ source A · p.12            │
│ ─ 텍스트      │ └────────────────────────┘     │   "원문 발췌…"               │
│ ─ 웹 URL      │                               │ ▸ source B · 03:20(영상)     │
│ ─ 영상 URL    │ [생성 옵션] 난이도·길이·톤      │                              │
│              │ [▶ 생성 실행]                  │ ⚠ 무출처 문장 3건            │
│ 소스 목록:    │ ────────────────────────       │   [재생성 권고]              │
│ ☑ 인덱싱완료  │ 생성 결과(본문 편집 영역)        │                              │
│ ⏳ 인덱싱중   │  · 문장마다 [¹][²] 인용 마커     │ ── 스킬 태깅 ──              │
│ ⚠ 실패       │  · 클릭 시 ③ 패널 하이라이트     │ + 스킬 매핑(택소노미 검색)    │
│              │                               │                              │
└──────────────┴───────────────────────────────┴──────────────────────────────┘
 하단 액션바: [임시저장]  [출처 검증]  [검수 요청 →]      자동저장 표시
```

검수 모드(4패널, 검수자 진입 시): ② 워크스페이스 우측에 **④ 검수 패널**(코멘트 스레드·승인/반려·버전 diff)이 추가로 열림. 출처 패널과 탭 전환 또는 분할.

상호작용 핵심:
- 본문의 인용 마커 ↔ 출처 패널 항목이 양방향 하이라이트(NotebookLM급 그라운딩 UX, 02-STRATEGY 해자 ③).
- 무출처 문장은 시각적 플래그 + 검수 제출 게이트.

| 앱 | GNB(상단) | 사이드바 / 컨텍스트 내비 |
|---|---|---|
| studio | 로고 · 프로젝트 선택 ▾ · [생성/출처/검수] 탭 · 상태뱃지 · 사용자 ▾ | (프로젝트 진입 시) 소스 / 워크스페이스 / 검수 / 설정 / 발행 |

내비 원칙:
- studio는 **프로젝트 중심**(프로젝트를 고른 뒤 단계 탭으로 이동) — 단계 탭이 곧 시나리오 §3의 흐름.
- admin은 **관리 도메인 중심**(좌측 사이드바 도메인 전환).
- lms는 **학습 여정 중심**(카탈로그 → 코스 → 수강 → 내 학습).

## 7. MVP 화면 우선순위 (가정)

02-STRATEGY §6(약점 3지점: 그라운딩·멀티모달 통합·표준 패키징)과 §10(산출물 축소: 요약·슬라이드·퀴즈 우선)에 정렬.

| 우선 | 화면 | 근거(전략 연결) |
|---|---|---|
| P0 | studio: 프로젝트 생성 · 소스 패널 · **생성 워크스페이스(3패널) · 출처 패널** | 멀티모달 통합 + 그라운딩 — 핵심 차별 데모 |
| P0 | studio: 산출물 편집기(요약·슬라이드·퀴즈 3종) | MVP 산출물 범위 축소 |
| P0 | studio: 검수(HITL) 화면 + 상태머신 | 해자 ④, 신뢰 메시지 완성 |
| P0 | studio: 발행(SCORM/xAPI + 네이티브) | 해자 ①, 표준 패키징 차별점 |
| P1 | lms: 카탈로그 · 수강 플레이어 · 평가 | 네이티브 발행의 도착지(엔드투엔드 증명) |
| P1 | admin: 콘텐츠/배포 · 회원/권한 | 발행→배포 연결 최소 운영 |
| P2 | admin: 스킬 택소노미 관리 + studio 스킬 태깅 | 스킬은 MVP에선 "씨앗"(수동), 본격 개인화는 후속 |
| P2 | admin: 과금 · 사용량 · 운영 설정(벤더 연동) | 상용화 단계 |
| P3 | 8종 풀세트(영상·오디오·커리큘럼·콘티·문제은행) · 자동 태깅 · 생성형 변주 | Phase 2(02-STRATEGY §10) |

MVP 한 줄: **studio의 P0 수직 슬라이스(소스→그라운딩 생성→검수→표준 발행)** 를 먼저 완성해 차별화 데모를 확보하고, lms·admin은 엔드투엔드 증명에 필요한 최소만 붙인다.

## 8. 미정 · 가정

- (가정) 형제 레포 미구현 → 본 IA는 그린필드. 구현 착수 시 실제 라우팅·컴포넌트 구조와 정합성 재검증.
- (가정) 역할은 기능 축별 분리, 4-eyes(자가검수 금지) 적용 — 조직 정책으로 완화 가능 여부 미정.
- (가정) 학습자에게 출처(citation)를 노출할지 — 신뢰 UX 강점이나 정보 과다 우려, A/B 검증 대상.
- (가정) 생성 워크스페이스 기본 3패널 / 검수 시 4패널 — 화면 폭·반응형 분기 기준 미정.
- (미정) 스킬 택소노미 가져오기 포맷(CSV/표준 스킬 프레임워크)과 자동 태깅 사전 출처.
- (미정) admin 배포 화면이 lms 코스 편성을 직접 소유할지, lms가 별도 코스 빌더를 가질지 경계.
- (미정) 산출물 버전·diff 비교의 단위(문서 전체 vs 문장 vs 슬라이드).
