# 01 — 학습목표 작성 규칙 + 블룸 택사노미

> 범위: 학습목표(learning_objective)의 작성 규칙, 블룸 택사노미 6수준 정렬, 난이도·계열화 원칙, 학습목표를 단일 앵커로 한 일관 파생 규칙.
> 정본 정합: 05-GENERATION §4.1(앵커)·§4.2(블룸)·§4.4(난이도). 데이터 모델: 05-GENERATION §6.3 `learning_objective`.
> 출처: 블룸 택사노미는 Anderson, L. W. & Krathwohl, D. R. (2001) 개정판(인지 영역 6수준: Remember/Understand/Apply/Analyze/Evaluate/Create)을 기준으로 한다(원판: Bloom et al., 1956). 출처: Wikipedia "Bloom's taxonomy" 정리, 2001 개정판.

---

## 1. 왜 학습목표가 먼저인가

학습목표는 **모든 산출물의 단일 앵커**다(05-GENERATION §4.1). 커리큘럼·요약·슬라이드·콘티·퀴즈·문제은행·영상·오디오가 모두 `objective_ref`로 학습목표를 참조한다. 따라서 학습목표의 품질이 무너지면 8종 전체의 교육 타당성이 함께 무너진다. **0단계는 항상 학습목표 도출**이며, 사용자가 직접 입력하거나 소스에서 LLM이 후보를 제안 → 사용자 확정한다.

---

## 2. 학습목표 작성 규칙

### 2.1 measurable 원칙 (필수)

학습목표는 **관찰·측정 가능**해야 한다. 학습 후 학습자가 **무엇을 할 수 있는지**를 행위동사로 진술한다.

- **권장 형식**(05-GENERATION §4.1): `행위동사(블룸) + 내용 + 조건/기준`
  - 예: "주어진 매출 데이터에서 피벗테이블을 **작성하여** 월별 추이를 요약할 수 있다."
- **반려 대상**(측정 불가): "~을 안다", "~을 이해한다", "~에 익숙해진다", "~을 배운다" 등 내적 상태 동사. 관찰 불가 → 평가 설계 불가 → 반려.

### 2.2 ABCD 모델 (작성 보조)

학습목표 진술을 점검하는 4요소 (출처: Mager 류 행동목표 + ABCD 모델, 교수설계 일반 프레임워크 / 일부 적용 정도는 가정).

| 요소 | 의미 | 필수도 | 예시 |
|---|---|---|---|
| **A**udience(대상) | 누가 — 학습자 | 묵시적 허용("학습자는" 생략 가능) | (학습자는) |
| **B**ehavior(행동) | 관찰 가능한 행위동사 + 내용 | **필수** | 피벗테이블을 작성하여 추이를 요약 |
| **C**ondition(조건) | 어떤 상황·자료·도구로 | 권장 | 주어진 매출 데이터에서 |
| **D**egree(기준) | 어느 수준·정확도로 | 권장(평가 채점선과 연결) | 정확히 / 3개 이상 / 80% 이상 |

- **B는 항상 필수**, C·D는 평가 채점 기준선과 직결되므로 가능한 한 명시 권장.
- D(기준)는 `02` 평가 정합의 채점 루브릭과 일관되어야 한다.

### 2.3 단일 행위 원칙

- 하나의 학습목표에는 **하나의 핵심 행위동사**만 둔다. "설명하고 적용한다"처럼 두 인지수준을 묶지 않는다(블룸 정합·평가 매핑이 모호해짐). 둘 다 필요하면 **목표를 분리**한다.

## 3. 블룸 택사노미 6수준 × 동사·평가형식 표

> 출처: Anderson & Krathwohl(2001) 개정 인지 영역 6수준. 퀴즈 문항 유형 매핑은 05-GENERATION §4.2 표를 따르되 본 문서에서 평가형식·자동채점 가능성을 보강한다.

| 수준 | 영문 | 인지 행위 | 대표 행위동사(한국어) | 권장 평가형식 | 자동채점 |
|---|---|---|---|---|---|
| 1 기억 | Remember | 사실·용어 회상·재인 | 정의하다, 나열하다, 식별하다, 명명하다, 매칭하다 | 단답, 진위(O/X), 용어 매칭 | 가능 |
| 2 이해 | Understand | 의미 파악·요약·분류 | 설명하다, 요약하다, 분류하다, 비교(단순)하다, 예시를 들다 | 객관식(개념 구분), 단답 | 가능 |
| 3 적용 | Apply | 새 상황에 절차·원리 적용 | 적용하다, 계산하다, 실행하다, 해결하다, 작성하다 | 사례형 객관식, 계산형 단답 | 대체로 가능 |
| 4 분석 | Analyze | 구성요소·관계 분해 | 비교·대조하다, 구분하다, 분해하다, 관계를 밝히다 | 다중정답형, 시나리오 분석, 매칭 | 부분 가능 |
| 5 평가 | Evaluate | 기준에 따른 판단·비판 | 판단하다, 비판하다, 정당화하다, 평가하다, 권고하다 | 근거 선택형, 서술형(기준 채점) | **제한적**(서술형은 HITL) |
| 6 창조 | Create | 새 산출물 설계·생성 | 설계하다, 작성(창작)하다, 제안하다, 구성하다, 개발하다 | 서술형, 과제·수행평가 | **불가(HITL 필수)** |

### 3.1 자동채점 경계 (중요)

- **블룸 1~3수준**: 선택형·단답형으로 자동채점 가능 → 퀴즈·문제은행 자동 출제에 적합.
- **블룸 4수준**: 형식에 따라 부분 가능(다중정답·매칭은 가능, 개방형 분석은 제한).
- **블룸 5~6수준**: 서술형·과제형이 본질 → **자동채점 한계, HITL 의존**(05-GENERATION §4.2). 루브릭·체크리스트에서 "사람 채점 필요"로 분기한다.

## 4. 난이도·계열화 원칙

### 4.1 난이도 척도 (파라미터화)

- 난이도 척도는 **하드코딩하지 않고 파라미터로** 둔다. 척도 확정은 미정(05-GENERATION §8): `easy / medium / hard` 3단계 또는 1~5 스케일(가정).
- 생성 시 **목표 난이도 분포**를 파라미터로 지정(예: easy 30% / medium 50% / hard 20%, 가정). 이 분포는 후속 개인화(생성형 변주, 07)의 입력이 된다.
- **난이도 ≠ 블룸 수준**: 블룸은 인지 *종류*, 난이도는 문항·과제의 *체감 곤란도*다. 같은 블룸 수준에서도 난이도는 달라질 수 있다. 둘을 별도 메타로 태깅한다.

### 4.2 계열화(시퀀싱) 원칙

- **선수→후속 순서**: 하위 블룸(기억·이해)이 상위 블룸(적용 이상)의 선수 조건이 되도록 차시를 배열한다(백워드 디자인 + 선행조직자 원칙 / 적용 정도는 가정).
- **스킬 선수관계 연계**: 학습목표에 스킬이 태깅되면, 스킬 `prerequisite_of` 관계(07-SKILLS)와 계열화가 모순되지 않아야 한다. (협업: 07 담당과 교차 검토.)
- **숙련 레벨 정합**: 학습목표의 블룸/난이도는 스킬 숙련 4단계(가정: 1인지/2적용/3숙달/4전문, 07-SKILLS)와 정합한다. 대략적 매핑(가정): 기억·이해→인지(1), 적용→적용(2), 분석·평가→숙달(3), 창조→전문(4). **확정 매핑은 07 담당과 합의 필요(가정)**.

## 5. 학습목표를 앵커로 한 일관 파생 규칙

> 05-GENERATION §2.1·§3.1·§4.5 정합. 파생은 단방향: 학습목표 → 커리큘럼 → 슬라이드 → 콘티 → 영상/오디오.

### 5.1 앵커 참조 강제

| 산출물 | 학습목표 참조 방식 | 위반 시 |
|---|---|---|
| 커리큘럼 | 각 lesson에 `objective_ref` | 앵커 없는 차시 불허 |
| 요약 | 산출물에 `objective_refs` (key_point가 목표를 지지) | 목표 미연결 경고 |
| 슬라이드 | 덱이 `lesson_ref`를 통해 목표 간접 참조 | 목표 미연결 경고 |
| 콘티 | 슬라이드 경유 목표 참조 | 슬라이드 정합에 종속 |
| 퀴즈 | 각 item에 `objective_ref` | **앵커 없는 문항 불허**(평가 정합 핵심) |
| 문제은행 | 각 item에 `objective_ref` | 앵커 없는 문항 불허 |

- **규칙**: 모든 산출물은 `objective_ref`로 학습목표를 참조해야 한다. **앵커 없는 파생물은 발행 불가.**

### 5.2 일관성·stale 전파

- 학습목표 변경은 **전 산출물에 영향**(05-GENERATION §4.5). 상위 산출물(목표·커리큘럼) 변경 시 하위는 `parent_version` 불일치로 **stale(재생성 권장)** 플래그를 받는다.
- **stale 강제 수준은 미정**(05-GENERATION §8): "권장 표시"인지 "발행 차단"인지 planner와 합의 전까지는 **권장 표시(가정)**로 운용한다.

### 5.3 도출 시 블룸·난이도 태깅 강제

- 학습목표 확정 시 **블룸 레벨을 반드시 태깅**(`learning_objective.bloom_level`)한다. 태깅 누락 목표는 평가 정합(`02`) 검증 불가 → 미완성 처리.
- 난이도(`difficulty`)도 함께 부여하되, 척도는 파라미터(§4.1)를 따른다.

## 6. 좋은/나쁜 예시 (프롬프트 few-shot용 — api-developer 핸드오프)

| 판정 | 학습목표 진술 | 사유 |
|---|---|---|
| 나쁨 | "엑셀 함수를 안다." | 측정 불가 동사("안다"), 블룸 매핑 불가 → 반려 |
| 나쁨 | "피벗테이블을 이해하고 보고서를 만든다." | 두 행위·두 인지수준 혼합 → 분리 필요 |
| 좋음 | "주어진 매출 데이터에서 피벗테이블을 **작성하여**(적용) 월별 매출 추이를 요약할 수 있다." | B+C+D 충족, 블룸=적용, 측정 가능 |
| 좋음 | "두 마케팅 캠페인의 ROI를 **비교·분석하여**(분석) 더 효과적인 안의 근거를 제시할 수 있다." | 블룸=분석, 관찰 가능, 평가형식(시나리오) 연결 가능 |
