# 03 — 산출물 8종별 품질 루브릭

> 범위: 공통 품질 기준(5축) + 산출물 8종별 루브릭. 각 루브릭은 평가 기준 항목 · 척도 · 통과 기준(발행 게이트) · 그라운딩 요건을 갖는다.
> 정본 정합: 05-GENERATION §3(산출물 스키마)·§4(ID 정렬)·§7.1(MVP 우선순위). 출처(source_refs)는 04-INGEST-GROUNDING 위임 결과를 사용.
> MVP 표기: `[MVP P0]` = 요약·슬라이드·퀴즈 / `[P1]` = 커리큘럼·콘티·문제은행·오디오·영상.

---

## 1. 척도와 발행 게이트 정의

### 1.1 공통 척도

각 항목은 **3단계 척도**로 평가한다(점수 병기).

| 척도 | 점수 | 의미 |
|---|---|---|
| 미흡 | 0 | 기준 미달 — 수정 필요 |
| 보통 | 1 | 허용 가능하나 개선 여지 |
| 우수 | 2 | 기준 충실 |

### 1.2 발행 게이트(통과 기준) 원칙

- **하드 게이트(무조건 차단)**: 출처(source_refs) 누락 단위가 1개라도 있으면 발행 불가(04 하드 게이트). 사실 오류(정확성=미흡)도 차단.
- **소프트 게이트(검수자 판단)**: 정확성·그라운딩 외 항목이 "미흡"이면 원칙적으로 반려하되, 검수자가 경미하다고 판단하면 코멘트 후 승인 가능.
- **기본 통과선**: 모든 항목 ≥ 보통(1), 그리고 **정확성·그라운딩 = 우수(2)** 강제. 즉 정확성·출처는 타협 불가.

## 2. 공통 품질 기준 (5축) — 8종 전체 선적용

> 산출물별 루브릭(§3~)은 이 5축을 상속하고 산출물 특화 항목을 추가한다.

| 축 | 정의 | 미흡(0) | 우수(2) | 게이트 |
|---|---|---|---|---|
| **C1 정확성** | 사실·개념·수치가 옳고 소스와 일치 | 사실 오류·환각 존재 | 오류 없음, 소스와 정합 | **하드** |
| **C2 그라운딩/출처** | 산출 단위마다 source_refs ≥1, 출처가 실제 내용을 지지 | 출처 누락·부정확 매핑 | 모든 단위 출처 보유·정확 매핑 | **하드** |
| **C3 교육적 구조** | 학습목표 정렬, 논리적 계열, 인지부하 적정 | 목표 미연결·무질서 | 목표 정렬·명료한 구조 | 소프트 |
| **C4 난이도 적합** | 대상·블룸·난이도 분포 적정 | 수준 부적합 | 목표 블룸·난이도와 일치 | 소프트 |
| **C5 접근성·편향/포용성** | 대체텍스트·명료한 언어·차별/편향 없음 | 차별·편향·접근성 결함 | 포용적·접근 가능 | 소프트(차별·편향은 하드) |

### 2.1 C5 세부 (접근성 + 편향/포용성)

- **접근성**: 시각 자료에 대체텍스트(`alt`)·캡션, 색만으로 정보 전달 금지, 명료한 문장(출처: WCAG 일반 원칙 / 적용 정도는 가정).
- **편향/포용성**: 성·연령·인종·지역·장애에 대한 고정관념·차별 표현 배제, 예시·인물의 다양성. **명백한 차별·편향은 하드 차단.**

## 3. [MVP P0] 학습 요약 (summary) 루브릭

> 스키마: `{ title, abstract, key_points[], glossary[], source_refs[] }` (05-GENERATION §3 ①)

| 항목 | 척도(0/1/2) | 통과 기준 |
|---|---|---|
| 정확성(C1) | 사실 오류 → 0 | **2 강제** |
| 그라운딩(C2) | key_point별 source_refs | **2 강제** |
| 핵심 포착 | 소스 핵심 개념 누락/왜곡 정도 | ≥1 |
| 목표 정렬(C3) | 학습목표(`objective_refs`)와 요약 초점 일치 | ≥1 |
| 압축 적정 | 과도 축약(핵심 손실)/장황 | ≥1 |
| 용어 정의 | glossary 정확·필요 용어 포함 | ≥1 |

- **발행 게이트**: 정확성·그라운딩 = 2, 나머지 ≥1. key_point 중 source_refs 없는 항목 0개.

## 4. [MVP P0] 슬라이드 (slides) 루브릭

> 스키마: `{ deck_id, lesson_ref, slides[]: { layout, title, bullets[], speaker_note, visual_hint, source_refs[] } }` (05-GENERATION §3 ③)

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 정확성(C1) | — | **2 강제** |
| 그라운딩(C2) | 슬라이드별 source_refs | **2 강제** |
| 1슬라이드 1메시지 | 한 슬라이드에 메시지 과다 여부 | ≥1 |
| 인지부하(C3) | 불릿 수·텍스트 밀도 적정 | ≥1 |
| 차시-목표 정렬 | `lesson_ref` → 목표 일관 | ≥1 |
| 발표자 노트 | speaker_note 유용·정확 | ≥1 |
| 접근성(C5) | visual_hint에 대체텍스트 가능성, 색 의존 금지 | ≥1 |

- **발행 게이트**: 정확성·그라운딩 = 2, 나머지 ≥1. 모든 슬라이드 source_refs 보유.

## 5. [MVP P0] 퀴즈 (quiz) 루브릭

> 스키마: `{ quiz_id, lesson_ref, items[]: { stem, type, options[], answer, rationale, bloom, difficulty, objective_ref, source_refs[] } }` (05-GENERATION §3 ⑤)
> **평가 정합(`02`)의 최우선 적용 대상.**

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 정확성(C1) | 정답·해설 오류 → 0 | **2 강제** |
| 그라운딩(C2) | 문항별 source_refs | **2 강제** |
| 목표 정렬 | 모든 item에 유효 `objective_ref` | **2 강제**(앵커 없는 문항 불허) |
| 블룸 정합 | 문항 블룸 ≥ 목표 블룸(가정, `02` §3) | ≥1 |
| 문항 작성 품질 | 명료한 stem, 매력적 오답(distractor), 단서 누출 없음 | ≥1 |
| 정답·해설 | 정답 유일·명확, rationale가 오답근거 포함 | ≥1 |
| 난이도 태깅(C4) | difficulty·bloom 태깅 정확 | ≥1 |

- **발행 게이트**: 정확성·그라운딩·목표 정렬 = 2, 나머지 ≥1. **목표 커버리지 미커버 0건**(`02` §4). 블룸 5~6 문항은 자동채점 한계 → 서술형은 HITL 채점 플래그 필수.

### 5.1 문항 작성 세부 점검(객관식)

- 보기는 길이·문법이 균질, 정답만 두드러지지 않을 것. "모두/항상/결코" 등 절대어 단서 회피. 부정형 발문은 강조 표기. (출처: 문항 작성 일반 원칙 / 적용 정도는 가정.)

## 6. [P1] 커리큘럼 (curriculum) 루브릭

> 스키마: `{ modules[]: { title, objectives_ref[], lessons[]: { title, objective_ref, duration_min, summary } } }` (05-GENERATION §3 ②)

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 정확성(C1) | — | **2 강제** |
| 그라운딩(C2) | lesson별 source_refs | **2 강제** |
| 목표 커버 | 모든 학습목표가 차시에 매핑(`objective_ref`) | **2 강제** |
| 계열화(C3) | 선수→후속 순서, 블룸 누적 적정(`01` §4.2) | ≥1 |
| 모듈 응집 | 모듈 내 차시 주제 일관 | ≥1 |
| 소요시간 | duration_min 합리성 | ≥1 |

- **발행 게이트**: 정확성·그라운딩·목표 커버 = 2. 앵커 없는 차시 0개. 미배치 학습목표 0개.

## 7. [P1] 콘티(스토리보드) (storyboard) 루브릭

> 스키마: `{ scenes[]: { slide_ref, narration, on_screen_text, shot_hint, duration_sec, source_refs[] } }` (05-GENERATION §3 ④)

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 정확성(C1) | 내레이션 사실 정확 | **2 강제** |
| 그라운딩(C2) | scene별 source_refs(슬라이드 경유) | **2 강제** |
| 슬라이드 정합 | `slide_ref`와 내용 일치(stale 아님) | **2 강제** |
| 내레이션 품질(C3) | 구어체 명료, 분량-시간 정합 | ≥1 |
| 자막/화면(C5) | on_screen_text 가독·접근성 | ≥1 |
| 장면 흐름 | 전환·길이 자연스러움 | ≥1 |

- **발행 게이트**: 정확성·그라운딩·슬라이드 정합 = 2. 상위(슬라이드) stale 시 재생성 권장.

## 8. [P1] 문제은행 (item_bank) 루브릭

> 스키마: quiz item 필드 + `{ tags[], usage_scope, dedup_hash }` (05-GENERATION §3 ⑥)
> 퀴즈 루브릭(§5)을 상속하고 풀(pool) 특화 항목 추가.

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| (퀴즈 §5 전 항목 상속) | — | 동일 |
| 중복 제거 | `dedup_hash` 기반 중복 문항 없음 | ≥1 |
| 변별·변주 | 동일 목표 내 난이도·관점 다양 | ≥1 |
| 메타 태깅 | tags·usage_scope·blooms 정확 | ≥1 |
| 스킬 태깅 정합 | content_skill_tag 후보 적정(07 협업) | ≥1 |

- **발행 게이트**: 퀴즈 게이트 + 중복 0건. 대량 배치는 **표본 검수 + 자동 점검** 병행(검수 부담 완화).

## 9. [P1] 학습 오디오 (audio) 루브릭

> 입력: 콘티/요약 스크립트 → 외부 TTS 렌더(06-MEDIA 위임). 본 루브릭은 **스크립트의 교육 품질 + 산출 메타**만 평가(렌더 품질은 06 소관).

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 스크립트 정확성(C1) | — | **2 강제** |
| 그라운딩(C2) | 스크립트 구간별 source_refs | **2 강제** |
| 청취 적합성(C3) | 구어체·호흡·길이 적정 | ≥1 |
| 목표 정렬 | 스크립트가 목표 지지 | ≥1 |
| 접근성(C5) | (가정) 텍스트 트랜스크립트 동반 | ≥1 |

- **발행 게이트**: 스크립트 정확성·그라운딩 = 2. 렌더 품질·동기화는 06-MEDIA 게이트에 위임.

## 10. [P1] 학습 영상 (video) 루브릭

> 입력: 콘티 → 외부 영상 합성(06-MEDIA 위임). 본 루브릭은 **콘티(스크립트·화면지시)의 교육 품질 + 산출 메타**만 평가.

| 항목 | 척도 | 통과 기준 |
|---|---|---|
| 콘티 정확성(C1) | — | **2 강제** |
| 그라운딩(C2) | scene별 source_refs | **2 강제** |
| 교육적 구성(C3) | 도입-전개-정리, 인지부하 적정 | ≥1 |
| 자막·접근성(C5) | 자막 제공, 색·청각 단독 의존 금지 | ≥1 |
| 목표 정렬 | 영상이 차시 목표 충족 | ≥1 |

- **발행 게이트**: 콘티 정확성·그라운딩 = 2, 자막 제공 ≥1. 렌더 품질은 06-MEDIA 게이트에 위임.
