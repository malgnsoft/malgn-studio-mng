# 교수설계 품질 정본 패키지 (quality/)

> 작성: instructional-designer · 기준 시점 2026-06-18
> 목적: 맑은스튜디오 생성 엔진(05-GENERATION)의 **교육 품질 기준**이자 **HITL 검수의 기준**을 정본화한다. 교수설계(ID) 정렬은 맑은스튜디오 4대 해자 중 신뢰성 축의 첫 기둥이다(02-STRATEGY 해자 ③④, 05-GENERATION §1).
> 책임 경계: 이 패키지는 **기준·루브릭·정렬 설계(정책)**만 정의한다. 프롬프트/파이프라인 구현은 api-developer(malgn-studio-api), 검수·매트릭스 화면은 frontend, 정본 통합·WBS는 planner 소관이다.

---

## 1. 이 패키지는 무엇인가

생성 엔진이 만드는 8종 산출물이 **교육적으로 신뢰할 수 있는가**를 판정하는 단일 기준 묶음이다. "생성"은 이미 상향 평준화된 기본 기능이므로, 우리의 차별점은 **학습목표를 단일 앵커로 한 교수설계 정렬 + 출처 그라운딩 + 사람 검수**다(05-GENERATION §1). 이 패키지는 그중 **교수설계 정렬과 검수 기준**을 담당한다.

## 2. 세 가지 사용처

| 사용처 | 누가 | 무엇을 위해 |
|---|---|---|
| ① 생성 프롬프트 품질 기준 | api-developer | 산출물별 교육 품질 요구사항·예시를 프롬프트/체인에 주입(구현은 api). 루브릭 통과 항목을 시스템 제약·few-shot으로 반영 |
| ② HITL 검수 기준 | 검수자(SME)·frontend | 검수자가 승인/반려를 판단하는 체크리스트·반려 사유 분류. 검수 UI가 이 체크리스트·경고 규칙을 구현 |
| ③ QA 회귀 기준 | qa | 생성 품질·출처 정합·목표 커버리지의 회귀 테스트 기준선. 발행 게이트(통과 기준) 충족 여부 자동/수동 검증 |

## 4. 읽는 순서

1. **`01`** — 모든 기준의 출발점인 학습목표·블룸을 먼저 이해한다(앵커).
2. **`02`** — 목표가 평가로 어떻게 커버되는지(정합 매트릭스)를 본다.
3. **`03`** — 산출물별 품질 합격선(루브릭·발행 게이트)을 본다.
4. **`04`** — 검수자가 실제로 쓰는 체크리스트·흐름을 본다.

## 5. 정본 의존 관계

- **상위 정본**: `05-GENERATION.md`(생성 명세·ID 정렬 §4·HITL 상태머신 §5·데이터 모델 §6) — 이 패키지는 그 §4의 정책을 **운용 가능한 기준**으로 구체화한 것이다.
- `03-FEATURES.md` — MVP 산출물 3종(요약·슬라이드·퀴즈, P0) 우선순위. 본 패키지도 **이 3종을 우선** 표기한다.
- `04-INGEST-GROUNDING.md`(위임) — 출처(source_refs) 추출·청킹은 04 소관. 본 패키지는 "source_refs 없는 산출 단위는 발행 차단"이라는 **하드 게이트 결과만** 사용한다.
- `07-SKILLS-PERSONALIZATION.md` — 학습목표↔스킬·숙련 레벨 정합. 학습목표가 스킬 태깅의 공통 키(협업: 07 담당과 교차 검토).

## 6. 핵심 원칙(요약)

1. **학습목표는 단일 앵커** — 커리큘럼→슬라이드→콘티→영상이 모두 `objective_ref`로 학습목표를 참조해야 발행 가능(05-GENERATION §4.1).
2. **학습목표는 measurable** — 관찰·측정 가능한 행위동사(블룸) 기반. "~을 안다/이해한다"류는 반려.
3. **모든 목표는 평가 ≥1로 커버** — 미커버 목표가 0이어야 통과(백워드 디자인, 05-GENERATION §4.3).
4. **그라운딩이 교육 타당성에 우선** — 교육적으로 타당해도 출처(source_refs) 없는 산출 단위는 무조건 발행 차단(하드 게이트, 04 정본).
5. **HITL 없이는 발행 없다** — published는 반드시 approved를 거치며, 자가검수 금지(4-eyes 원칙, 단 MVP 1인 체제 예외는 `04` 참조).

## 7. MVP 범위 표기

- 본 패키지는 **8종 전체** 기준을 담되, **요약·슬라이드·퀴즈(P0)**를 우선 적용 대상으로 표기한다(루브릭·체크리스트에 `[MVP P0]` / `[P1]` 라벨).
- 평가 정합 매트릭스(`02`)의 최우선 적용 대상은 **퀴즈** — 평가-목표 정렬(차별화)을 가장 잘 드러내는 산출물(05-GENERATION §7.1).
