# 차별화 전략 — 맑은스튜디오 (LMS AI Studio)

> 목적: 경쟁 환경 분석(01-COMPETITION.md)을 근거로 맑은스튜디오의 차별화 전략·해자·속도 전략을 확정한다.
> 정본 참조: 본 문서의 모든 판단은 01-COMPETITION 정본을 근거로 하며, 추정은 "(가정)"으로 표기한다.

---

## 1. 목적

- "AI 콘텐츠 생성"은 이미 상향 평준화된 기본 기능이다(01-COMPETITION §0). 생성만으로는 차별화 불가.
- 본 문서는 **무엇으로 이기고, 무엇을 만들지 않을지**를 결정한다.
- 핵심 결론: 우리의 승부처는 단일 기능이 아니라 **4대 해자의 결합**, 그리고 **휴넷이 고도화를 끝내기 전에 약점 지점을 선점하는 속도**다.

---

## 2. 핵심 결정 / 접근

- **결정 A — 생성은 평준화 영역으로 보고 과투자하지 않는다.** 8종 산출물 생성은 "충분히 좋게", 차별화 투자는 해자 4종에 집중.
- **결정 B — 영상/오디오는 Buy(외부 API 연동).** 직접 개발하지 않는다(01-COMPETITION §1.1, §2.3). 우리는 발행·개인화·신뢰성에 자원을 투입.
- **결정 C — 해자는 결합으로만 성립한다.** 그라운딩 단독, 스킬 단독은 글로벌 강자가 이미 보유. 4종을 "자사 LMS 네이티브 발행"으로 묶는 결합이 국내 빈자리.
- **결정 D — 속도 우선.** MVP는 휴넷이 약한 3개 지점(그라운딩 신뢰성·멀티모달 통합·표준 패키징)에 집중해 차별화 메시지를 먼저 점유.

## 4. 주요 플로우 / 논리 — 4대 해자 심화

### 해자 ① 자사 LMS 네이티브 발행 + 표준 패키징(SCORM/xAPI)

- **왜 해자인가**: 생성 도구(브루·Synthesia)와 올인원 빌더(Coursebox)는 자체 LMS에 종속되거나 발행이 약하다. 우리는 자사 LMS(malgn-studio-lms)로 즉시 네이티브 발행 + SCORM/xAPI로 **타사 LMS로도 내보내기** → 종속 없이 양면 공략.
- **경쟁 대비 우위**: 글로벌 올인원은 자체 LMS 종속(나가기 어려움), 국내 생성도구는 발행 자체가 없음. 표준 패키징 동시 제공은 국내에서 비어 있음.
- **구현 난이도**: 중. 자사 LMS 연동은 보유 자산 활용으로 낮음. SCORM 1.2/2004·xAPI 패키저는 표준 스펙 구현 부담(중).

### 해자 ② 스킬 기반 개인화(스킬 택소노미 + 적응형 + 생성형 변주)

- **왜 해자인가**: 스킬 택소노미/스킬맵은 엔터프라이즈 LXP의 **기본 입장권**(Docebo·Cornerstone, 01-COMPETITION §1.3). 없으면 LXP 경쟁에서 탈락. 차별점은 큐레이션 추천을 넘어 **생성형 변주**(같은 스킬을 학습자 수준별로 재생성).
- **경쟁 대비 우위**: 경쟁사는 기존 콘텐츠 큐레이션 추천 위주. 우리는 그라운딩된 소스에서 난이도·형식을 **변주 생성** → 개인화 깊이 차별화.
- **구현 난이도**: 높음. 스킬 택소노미 구축 + 산출물-스킬 태깅 + 적응형 로직 + 변주 생성 파이프라인. (가정) 기존 LXP에 스킬 프로파일이 없으면 구축이 선행 과제.

### 해자 ③ 소스 그라운딩 · 출처 표기(NotebookLM급, 환각 차단)

- **왜 해자인가**: 국내 AI LMS는 대부분 요약·자막·문제 생성에 머물고 **출처 표기가 약하다**(01-COMPETITION §3-2). "업로드 자료 기반 + 출처 매핑으로 환각 차단"을 전면에 내세운 곳이 거의 없음 → 교육 신뢰성이라는 빈 메시지 점유.
- **경쟁 대비 우위**: NotebookLM·Sana가 글로벌 기준점이나 국내 통합 빌더엔 부재. 그라운딩을 8종 산출물 전체에 적용하면 강력한 신뢰 서사.
- **구현 난이도**: 중~높음. RAG 인덱싱 + 출처 인용(citation) UX + 산출물별 근거 추적. 산출물마다 출처를 거는 일관성 확보가 난점.

### 해자 ④ 사람 검수(HITL) 품질 보증

- **왜 해자인가**: 시장은 자동 생성 위주(01-COMPETITION §4). 교육 콘텐츠는 오류 비용이 크다. **검수 게이트 없이는 신뢰 마케팅이 공허**해진다. 해자 ③(그라운딩)과 결합해 "출처 있는 초안 → 사람 승인" 흐름으로 신뢰를 완성.
- **경쟁 대비 우위**: 자동 생성 일변도 경쟁사 대비 "품질 보증된 발행"이라는 차별점. B2B/공공 구매 설득에 강함.
- **구현 난이도**: 낮음~중. 상태머신(초안→검수중→승인/반려) + 코멘트·버전. 기능 자체는 단순하나 8종 산출물 전반의 워크플로 통합이 관건.

## 6. 속도 전략 — 휴넷이 약한 지점 선점 논리

- **전제**: 휴넷은 LMS('러닝메이커')·콘텐츠·영업력을 갖춘 채 **연내 AI LMS 고도화 예고**(01-COMPETITION §2.1) → 정면 기능 경쟁은 불리, **시간 싸움**.
- **논리**: 휴넷이 강한 곳(LMS·영업)을 피하고, 약한 3개 지점에 MVP를 집중해 차별화 메시지를 먼저 점유한다.

| 휴넷 약점 지점 | 왜 약한가 | 우리 MVP 집중 |
|---|---|---|
| 그라운딩 신뢰성 | 출처 표기 약함(국내 공통) | 8종 산출물 출처 인용 전면화 |
| 멀티모달 통합 | APL은 진단·개인화 축, 통합 생성 빌더 아님 | 단일 소스 → 8종 산출물 원스톱 |
| 표준 패키징 | 자체 LMS 강하나 표준 내보내기 약함(가정) | SCORM/xAPI 발행으로 락인 회피 양면 공략 |

- **속도 보조 수단**: 영상/오디오 Buy(직접 개발 생략), 브루 등 국산 영상도구 **연동/제휴**(01-COMPETITION §3-3)로 개발 시간 단축.

## 7. Build vs Buy 결정표

| 영역 | 결정 | 근거 |
|---|---|---|
| AI 영상 생성(아바타) | **Buy** | Synthesia/HeyGen/Colossyan/브루 강함, 직접 개발 비합리(§1.1) |
| AI 오디오(TTS/오디오 오버뷰) | **Buy** | 외부 TTS·음성 API 성숙, 차별 요소 아님 |
| 소스 그라운딩 / 출처 표기 | **Build** | 핵심 해자 ③, 8종 산출물 일관 적용은 직접 제어 필요 |
| 생성 오케스트레이션(8종 파이프라인) | **Build** | 통합·교수설계 정렬이 차별점, 우리 IP |
| 발행 / 표준 패키징(SCORM·xAPI) | **Build** | 핵심 해자 ①, 자사 LMS 자산 활용 |
| 스킬 택소노미 / 개인화 | **Build** | 핵심 해자 ②, 입장권+생성형 변주 차별화 |
| HITL 검수 워크플로 | **Build** | 핵심 해자 ④, 단순하나 우리 품질 게이트 |
| 문서/PPT/웹/영상 파싱(인제스트) | **Buy/Hybrid** (가정) | 파서·추출은 라이브러리·API 활용, 통합만 Build |
| LLM(생성 모델) | **Buy** | 모델은 외부 API, 프롬프트·오케스트레이션만 Build |

> 한 줄 원칙: **차별화(해자)는 Build, 평준화·인프라는 Buy.**

## 10. 미정 · 가정

- (가정) 기존 LXP에 스킬 프로파일/택소노미 보유 여부 미확인 → 확인 시급(01-COMPETITION §3-4).
- (가정) 우선 타깃 세그먼트 순서(기업교육 1순위)는 영업 데이터로 재검증 필요.
- (미정) 영상/오디오 1차 연동 벤더 선정(Synthesia vs 브루 vs 다중).
- (미정) SCORM 버전 우선순위(1.2 vs 2004) 및 xAPI LRS 연동 범위.
- (미정) 그라운딩 RAG 스택(임베딩·벡터스토어) 선택.
