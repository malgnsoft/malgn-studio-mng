# AI 생성 엔진 — 산출물 8종 생성 명세

> 범위: 맑은스튜디오 AI 생성 파이프라인. LLM 기반 산출물 6종(요약·커리큘럼·슬라이드·콘티·퀴즈·문제은행)의 생성 명세, 교수설계 정렬, HITL 검수 게이트, 데이터 모델.
> 위임: 영상·오디오 생성은 `06-MEDIA.md`(외부 API 연동). 소스 그라운딩/출처 추출의 상세는 `04-INGEST-GROUNDING.md`. 본 문서는 "출처를 받아 생성에 주입"하는 연결점만 다룬다.
> 정본 참조: `01-COMPETITION.md` (해자 = 교수설계 정렬 + 소스 그라운딩 + HITL).
> 구현 영역: 백엔드·파이프라인은 `malgn-studio-api`.

---

## 1. 목적

- AI 생성은 시장에서 이미 기본 기능(상향 평준화). 따라서 본 엔진의 목표는 **"생성"이 아니라 "교육적으로 신뢰할 수 있는 산출물"**이다.
- 세 가지 품질 축을 모든 산출물에 일관 적용한다:
  1. **교수설계(ID) 정렬** — 학습목표 기반, 블룸 택사노미, 평가-목표 정렬, 난이도 통제.
  2. **소스 그라운딩** — 모든 생성 문장은 인입된 출처(청크)에 연결되어 환각을 차단(상세는 `04`).
  3. **HITL 검수 게이트** — 초안은 사람 검토·승인 전까지 발행 불가.
- 8종 산출물이 **하나의 학습목표 집합에서 파생**되어 상호 일관성을 갖도록 한다(커리큘럼 → 슬라이드 → 콘티 → 영상).

## 2. 접근

### 2.1. 생성 파이프라인 전체 흐름

```
[인입·그라운딩(04)]
   소스(문서/PPT/웹/영상) → 청크 + 임베딩 + 출처 인덱스(grounding_index)
        │
        ▼
[1단계: 학습목표 도출]  소스 + (선택)사용자 의도 → learning_objective[]  ← ID 정렬의 앵커
        │
        ▼
[2단계: 커리큘럼 생성]  목표 → 모듈/차시 구조(curriculum)
        │
        ├──► 학습 요약(summary)        ── 목표·핵심 개념 압축
        ├──► 슬라이드(slides)          ── 차시별 슬라이드 덱
        │        └──► 콘티(storyboard) ── 슬라이드 + 내레이션 스크립트
        │                 └──► 영상/오디오 (06-MEDIA, 외부 API)
        ├──► 퀴즈(quiz)                ── 형성평가, 목표 커버리지
        └──► 문제은행(item_bank)        ── 대량 문항 풀, 메타데이터 태깅
        │
        ▼
[3단계: HITL 검수 게이트]  draft → review → approved → published
```

### 2.2. 생성 공통 원칙

- **RAG-그라운딩 강제**: 프롬프트에 검색된 출처 청크를 주입하고, "주어진 출처에 없는 사실은 생성 금지" 시스템 제약. 각 산출물 단위(슬라이드 1장, 문항 1개)마다 `source_refs[]`(청크 ID 목록)를 함께 출력하도록 스키마에 강제.
- **구조화 출력(JSON)**: 모든 LLM 산출물은 JSON 스키마로 강제(function calling / structured output). 편집 UI와 발행 패키징(SCORM/xAPI)이 구조에 의존.
- **재현·감사 가능성**: `generation_job`에 모델·프롬프트 버전·소스 스냅샷·파라미터를 기록 → 재생성·디버깅·품질 추적.
- **모델 라우팅(가정)**: 고난도 추론(목표 도출·커리큘럼 설계·문항 변별)은 상위 모델, 단순 변환(요약·슬라이드 텍스트화)은 비용 효율 모델로 라우팅. 모델 선정은 `claude-api` 기준 별도 검토.

## 3. 산출물별 생성 명세 표

> 영상·오디오는 본 표에서 제외(06-MEDIA 위임). 아래 6종은 모두 LLM 생성.

| # | 산출물 | 입력 | 모델/프롬프트 전략 | 출력 구조(스키마 핵심) | 편집 가능 요소 | 출처 연계 |
|---|---|---|---|---|---|---|
| ① | **학습 요약** (summary) | 소스 청크 + 학습목표 | Map-Reduce 요약(청크별 요약→통합), 목표 정렬 리랭킹. 길이·관점(개요/심화) 파라미터화 | `{ title, abstract, key_points[], glossary[], source_refs[] }` | 핵심 포인트 추가/삭제/재배열, 용어 정의 수정, 길이 재생성 | 각 key_point에 `source_refs`(청크 ID) |
| ② | **커리큘럼** (curriculum) | 학습목표[] + 소스 범위 | 백워드 디자인 프롬프트(목표→평가→내용 순). 모듈/차시 트리 생성, 차시별 목표 매핑·소요시간 추정 | `{ modules[]: { title, objectives_ref[], lessons[]: { title, objective_ref, duration_min, summary } } }` | 모듈·차시 추가/삭제/순서, 차시-목표 매핑, 소요시간 | 각 lesson에 `objective_ref` + `source_refs` |
| ③ | **슬라이드** (slides) | 차시(커리큘럼 노드) + 소스 | 차시 1개 → 슬라이드 덱. 슬라이드당 1메시지 원칙, 발표자 노트 생성. 레이아웃 타입 태깅(표지/본문/요약/평가) | `{ deck_id, lesson_ref, slides[]: { layout, title, bullets[], speaker_note, visual_hint, source_refs[] } }` | 슬라이드 추가/삭제/병합, 불릿 텍스트, 발표자 노트, 레이아웃, 비주얼 힌트 | 슬라이드별 `source_refs` |
| ④ | **콘티(스토리보드)** (storyboard) | 슬라이드 덱 | 슬라이드 → 장면(scene) 변환. 슬라이드별 내레이션 스크립트 + 화면 지시(샷·자막·전환) 생성. 영상 입력 규격에 맞춤 | `{ board_id, deck_ref, scenes[]: { slide_ref, narration, on_screen_text, shot_hint, duration_sec, source_refs[] } }` | 내레이션 스크립트, 자막, 장면 길이, 샷 지시, 장면 순서 | scene별 `source_refs`(슬라이드 경유) |
| ⑤ | **퀴즈** (quiz) | 학습목표[] + 차시 내용 | 목표별 1+문항(평가-목표 정렬). 블룸 레벨·난이도 지정 생성. 정답·오답근거·해설 동시 생성. 형성평가용(차시 직결) | `{ quiz_id, lesson_ref, items[]: { stem, type, options[], answer, rationale, bloom, difficulty, objective_ref, source_refs[] } }` | 문항 추가/삭제, 보기·정답·해설 수정, 난이도·블룸 재지정, 재생성 | 문항별 `objective_ref` + `source_refs` |
| ⑥ | **문제은행** (item_bank) | 학습목표[] + 소스 전체 | 목표·개념 단위 대량 문항 생성(배치). 변별도 고려 변주, 중복 제거, 메타데이터 자동 태깅. 형성/총괄 모두 풀(pool)로 적재 | `{ bank_id, items[]: { …quiz item 필드…, tags[], usage_scope, dedup_hash } }` | 문항 일괄 검토·승인/반려, 태그 수정, 풀에서 퀴즈로 차출 | 문항별 `objective_ref` + `source_refs` |

> 영상(⑦)·오디오(⑧)는 콘티(④)/요약(①)을 입력으로 `06-MEDIA.md`의 외부 API 렌더 잡으로 처리. 본 엔진은 그 입력 산출물(스크립트·슬라이드)까지 책임지고, 렌더는 위임한다.

## 4. 교수설계(ID) 정렬

### 4.1. 학습목표(learning_objective)를 앵커로

- 모든 생성의 **0단계는 학습목표 도출**. 사용자가 직접 입력하거나, 소스에서 LLM이 후보를 제안→사용자 확정.
- 학습목표는 **행위동사(블룸) + 내용 + 조건/기준** 형식 권장(예: "주어진 사례에서 OO 원리를 *적용*하여 결과를 예측할 수 있다").
- 이후 커리큘럼·슬라이드·퀴즈·문제은행이 모두 `objective_ref`로 이 목표를 참조 → 산출물 일관성의 기준선.

### 4.2. 블룸 택사노미(Bloom's Taxonomy) 적용

| 블룸 레벨 | 동사 예시 | 퀴즈 문항 유형 매핑(가정) |
|---|---|---|
| 기억(Remember) | 정의, 나열, 식별 | 단답·진위·용어 매칭 |
| 이해(Understand) | 설명, 요약, 분류 | 객관식(개념 구분) |
| 적용(Apply) | 적용, 계산, 실행 | 사례형 객관식·단답 |
| 분석(Analyze) | 비교, 구분, 분해 | 다중정답·시나리오 |
| 평가(Evaluate) | 판단, 비판, 정당화 | 서술형·근거 선택 |
| 창조(Create) | 설계, 작성, 제안 | 서술형·과제형(자동채점 한계 → HITL) |

- 학습목표마다 블룸 레벨을 태깅하고, 퀴즈·문제은행 문항도 블룸 레벨을 갖는다 → **목표-평가의 인지수준 정렬** 검증 가능.

### 4.3. 평가-목표 정렬(커버리지)

- **모든 학습목표는 최소 1개 이상의 평가 문항으로 커버**되어야 한다(백워드 디자인).
- 시스템이 `objective × item` 커버리지 매트릭스를 산출 → 검수 UI에 "미커버 목표" 경고.
- 블룸 레벨 불일치 경고(예: 목표는 '적용'인데 문항은 '기억' 수준만 존재) → 재생성 유도.

| 검증 항목 | 규칙 | 위반 시 |
|---|---|---|
| 목표 커버리지 | 목표당 평가 문항 ≥ 1 | 경고 + 문항 자동 제안 |
| 블룸 정합 | 문항 블룸 ≥ 목표 블룸(가정) | 경고 |
| 출처 그라운딩 | 산출 단위당 source_refs ≥ 1 | 발행 차단(하드 게이트) |

### 4.4. 난이도 통제

- 난이도 척도(가정): `easy / medium / hard` 또는 1~5 스케일. 문항·차시 단위로 부여.
- 생성 시 목표 난이도 분포를 파라미터로 지정(예: easy 30% / medium 50% / hard 20%).
- 후속(LXP 개인화)에서 적응형 출제·생성형 변주의 입력으로 사용(`01` 스킬 개인화 연계).

### 4.5. 산출물 간 일관성 파이프라인

- **커리큘럼 → 슬라이드 → 콘티 → 영상**이 단방향 파생되므로, 상위 산출물 변경 시 하위는 **재생성 필요(stale)** 플래그를 받는다.
- 일관성 보장 장치:
  - 하위 산출물은 상위의 버전(`parent_version`)을 기록 → 상위 변경 감지 시 "재생성 권장" 표시.
  - 학습목표 변경은 전 산출물 영향 → 영향 범위 시각화(가정).

## 5. HITL 검수 게이트

### 5.1. 상태 모델

```
                ┌──────────────── 재생성/변주 (regenerate) ───────────────┐
                ▼                                                          │
  (생성요청) → [generating] → [draft] ──검토요청──► [in_review] ──┬─ 승인 ─► [approved] ─발행─► [published]
                  │  실패         │                                 │
                  ▼              │ 직접수정                          └─ 반려(수정요청) ─► [draft]
              [failed]           ▼                                                    (사유 기록)
              (재시도)        [draft] (편집 반영)                    [published] ──개정──► [draft] (새 버전)
```

| 상태 | 의미 | 가능 액션 |
|---|---|---|
| `generating` | 생성 잡 진행 중 | 취소 |
| `failed` | 생성 실패 | 재시도, 파라미터 조정 |
| `draft` | 초안(편집 가능) | 직접 수정, 재생성, 변주, 검토 요청 |
| `in_review` | 검수자 검토 중 | 승인, 반려(사유), 코멘트 |
| `approved` | 승인됨(발행 대기) | 발행, 검토로 되돌리기 |
| `published` | 발행됨(LMS 노출) | 개정(새 draft 버전 파생) |

- **하드 게이트**: `published`는 반드시 `approved`를 거친다(자동 승인 없음, MVP 정책 / 가정: 후속에 신뢰도 높은 산출물 자동승인 옵션 검토).
- **출처 게이트**: source_refs 누락 산출 단위가 있으면 승인 불가.

### 5.2. 검수자 역할

| 역할 | 권한 | 책임 |
|---|---|---|
| 작성자(Author) | 생성·편집·재생성·검토요청 | 초안 품질, 출처 확인 |
| 검수자(Reviewer/SME) | 승인·반려·코멘트 | 내용 정확성, 교수설계 정합, 출처 타당성 |
| 발행자(Publisher) | 발행·개정 관리 | 발행 시점·버전 관리 |
| 관리자(Admin) | 역할·정책·게이트 설정 | 워크플로 거버넌스 |

> MVP에서는 작성자=검수자 1인 체제도 허용하되, 검토 단계는 강제(셀프 리뷰라도 명시적 승인 액션 필요).

### 5.3. 변주·재생성

- **재생성(regenerate)**: 동일 입력·다른 시드/파라미터로 산출물 전체 또는 일부(슬라이드 1장, 문항 1개) 다시 생성. 이전 버전 보존.
- **변주(variation)**: 난이도·관점·길이·톤을 바꿔 대안 생성(예: 같은 목표의 쉬운/어려운 문항). 개인화(생성형 변주, `01`)의 기반.
- 부분 재생성은 산출물 단위(item/slide/scene) 레벨로 지원하여 전체 폐기를 방지.

## 6. 데이터 모델 초안

> `malgn-studio-api` 구현 기준. 타입·제약은 (가정), 확정은 설계 단계에서.

### 6.1. 핵심 엔티티

```
content_project (콘텐츠 프로젝트)
```

## 7. MVP vs 후속

### 7.1. 산출물 우선순위

| 우선 | 산출물 | 근거 |
|---|---|---|
| **MVP** | 학습 요약 | 그라운딩 가치 즉시 입증, 구현 단순 |
| **MVP** | 커리큘럼 | 전 산출물의 구조 앵커, 교수설계 정렬 시연 |
| **MVP** | 퀴즈 | 평가-목표 정렬(차별화) 가장 잘 드러남 |
| **MVP** | 슬라이드 | 시각 산출물, 콘티·영상의 전 단계 |
| 후속 | 콘티(스토리보드) | 슬라이드 안정화 후, 영상 전 단계로 |
| 후속 | 영상·오디오 | 외부 API 연동(06), 비용·렌더 잡 안정화 후 |
| 후속 | 문제은행 | 대량 배치·중복제거·태깅 운영 부담 → 퀴즈 안정 후 |

> 근거: `01` 전략 — 그라운딩 신뢰성 + 교수설계 정렬을 빠르게 시연. 요약·커리큘럼·퀴즈·슬라이드가 이 가치를 가장 적은 외부 의존으로 보여준다. 영상은 외부 API 의존·비용 변수가 커 후속.

### 7.2. 기능 단계

| 영역 | MVP | 후속 |
|---|---|---|
| 그라운딩 | 출처 청크 주입 + source_refs 출력 | 출처 신뢰도 스코어·자동 검증 |
| ID 정렬 | 목표 도출 + 블룸 태깅 + 커버리지 경고 | 적응형 난이도·생성형 변주(개인화) |
| HITL | draft→review→approved→published, 1인 검수 허용 | 다단계 검수·자동승인 옵션·코멘트 협업 |
| 재생성 | 전체/부분 재생성 | 변주(대안 생성)·A/B |
| 발행 | (위임) LMS 발행·SCORM/xAPI 패키징 연계 | — |

## 8. 미정·가정

- (가정) 모델 라우팅·구체 모델은 `claude-api` 기준 별도 결정. 상위/효율 모델 2단 라우팅 가정.
- (가정) artifact는 JSONB payload 단일 테이블로 MVP 시작, 질의 부담 커지면 타입별 정규화로 분리.
- (가정) 블룸 정합 규칙("문항 블룸 ≥ 목표 블룸")은 교육 검토 후 확정.
- (가정) 자동 승인(신뢰도 높은 산출물)은 후속. MVP는 명시적 승인 강제.
- (미정) 난이도 척도(3단계 vs 1~5)와 차시별 소요시간 추정 방식.
- (미정) 산출물 간 stale 전파의 강제 수준(권장 표시 vs 발행 차단).
- (미정) 서술형·과제형 문항 자동채점 범위(블룸 상위는 HITL 의존).
- 위임: 출처 추출·청킹·임베딩 상세 → `04-INGEST-GROUNDING.md`. 영상·오디오 렌더 → `06-MEDIA.md`. LMS 발행·패키징 → 발행 문서(별도).
