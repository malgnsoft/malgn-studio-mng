# 비기능 요구사항(NFR) — 맑은스튜디오(Malgn Studio)

> 정본: 전략 [01-COMPETITION.md](./01-COMPETITION.md), 아키텍처 [09-ARCHITECTURE.md](./09-ARCHITECTURE.md).
> 각 항목은 **목표·전략·미정**을 표로 정리. 수치·구체 스택은 미정이며 추정은 "(가정)".

---

## [목적]

- NotebookLM급 신뢰성과 표준 발행을 내세우는 제품으로서, **기능보다 먼저 무너질 수 있는 비기능 축**(신뢰성·보안·개인정보·확장성·비용·성능·가용성·접근성·컴플라이언스)을 명시한다.
- 해자(그라운딩·HITL·표준 발행·스킬 개인화)와 직접 연결된 NFR을 우선순위화하고, 각 축의 **목표·전략·미정**을 합의 가능한 형태로 고정한다.

---

## [구성요소]

각 NFR 축을 표로 정리한다. (가정)은 추정, 미정은 결정 필요 항목.

### 1. 신뢰성 (그라운딩 정확도·출처 검증·품질) — **핵심 해자**

| 항목 | 목표 | 전략 | 미정 |
|---|---|---|---|
| 그라운딩 정확도 | 생성 주장이 입력 소스에 근거 (환각 차단) | RAG 근거 청크 한정 생성, 근거 없는 주장 거부, 출처 매핑 단계에서 citation 강제 | 근거 적중률·환각률 목표 수치 |
| 출처 검증 | 모든 핵심 주장에 추적 가능한 출처(`citation → source_chunk`) | 산출물 ↔ 소스 스팬 매핑, 출처 누락 시 발행 차단(가정) | 출처 커버리지 기준(전 주장 vs 핵심 주장) |
| 품질 보증 | 자동 생성물의 사람 검수 통과 | HITL 게이트(승인/반려), 반려 시 재생성, 반려율·재작업률 관측 | 산출물 종류별 자동 품질 점검 기준 |
| 정합성 | 멱등·재시도로 동일 입력→동일 산출 재현 | 멱등 키, 단계 체크포인트, 부분 재개 | 재현성 보장 범위(LLM 비결정성 한계) |

### 2. 보안 (업로드 자료 격리·저작권/IP·시크릿)

| 항목 | 목표 | 전략 | 미정 |
|---|---|---|---|
| 업로드 자료 격리 | 테넌트 간 소스·산출물 비공유 | `tenant_id` 행 격리 + 벡터 네임스페이스/스토리지 경로 격리, 권한 게이트 | 강격리(테넌트별 인덱스/버킷) 적용 범위 |
| 저작권/IP | 업로드 자료·생성물의 권리 관계 추적 | 소스 출처·라이선스 메타 보존, 발행물에 출처 표기, 외부 전송 범위 기록 | 업로드 권리 동의 모델, 생성물 저작권 귀속 정책 |
| 시크릿 | provider 키·토큰 비노출 | 시크릿 매니저 보관, 코드·프롬프트·로그·산출물에 미기입, 회전(rotation) | 키 회전 주기, 외부 의존별 자격 관리 방식 |
| 전송/저장 보호 | 전송·저장 구간 보호 | 전송 암호화·저장 암호화(가정), 접근 감사 로그 | 암호화 키 관리·감사 보존 기간 |

### 3. 개인정보 (학습자 데이터·PII·동의)

| 항목 | 목표 | 전략 | 미정 |
|---|---|---|---|
| 학습자 데이터 | 진도·평가·프로필 최소 수집·보호 | `learner_profile`/`xapi_statement` 최소수집, 목적 외 사용 금지 | 수집 항목·보존 기간 |
| PII | 소스·로그 내 PII 노출 최소화 | 외부 provider 전송 전 민감정보 점검, 로그 마스킹(가정) | PII 자동 탐지·마스킹 범위 |
| 동의 | 수집·외부 처리 동의 확보 | 업로드·학습 데이터 동의 플로우, 외부 AI 처리 고지 | 동의 철회·삭제 요청 처리 절차 |
| 외부 처리 경계 | LLM/미디어 API로 보낸 데이터 추적 | 테넌트 식별자 전파, 전송 콘텐츠·보존 추적, 데이터 보존 요건 점검 | provider별 데이터 보존/학습 사용 정책 확인 |

### 4. 확장성 (대용량 미디어·동시 생성 잡·큐)

| 항목 | 목표 | 전략 | 미정 |
|---|---|---|---|
| 대용량 미디어 | 대용량 영상·오디오 처리 | 외부 미디어 API 위임, 에셋 스토어 분리, 비동기 렌더 | 단일 산출물 최대 길이·용량 |
| 동시 생성 잡 | 다수 테넌트 동시 생성 흡수 | 잡 큐 + 워커 수평 확장, 단계별 잡 분리 | 동시 잡 처리량 목표 |
| 큐 안정성 | 폭주 시에도 유실·역류 없음 | 백프레셔·우선순위·재시도, 테넌트별 공정 분배(가정) | 큐 제품, 우선순위·격리 정책 |
| 인덱스 확장 | 소스 증가에도 검색 성능 유지 | 벡터 인덱스 분할·테넌트 네임스페이스 | 인덱스 샤딩 전략 |

### 5. 비용 (LLM/미디어 크레딧 관리·쿼터)

| 미디어 크레딧 | 외부 미디어 과금 통제 | 미디어 잡 멱등·중복 차단, 크레딧 차감 연동 | 벤더 과금 모델 |

## [데이터 흐름]

- NFR은 [09-ARCHITECTURE.md](./09-ARCHITECTURE.md)의 파이프라인·엔티티에 매핑된다:
  - 신뢰성 → `source_chunk`/`citation`/`hitl_review`, 파이프라인 출처 매핑·HITL 단계.
  - 보안·개인정보 → `tenant` 격리, 외부 provider 전송 경계, 시크릿 관리.
  - 확장성·성능 → `pipeline_run`/`pipeline_step` 잡 큐·워커, 벡터 인덱스.
  - 비용 → 잡별 토큰·비용 관측, 테넌트 크레딧/쿼터.
  - 가용성 → provider 추상화 폴백·재시도, 스토리지 내구성.
  - 접근성·컴플라이언스 → `package`/`course`/`xapi_statement` 발행·LRS·표준.

---

## [데이터 모델 통합]

- NFR 측정·집행은 통합 데이터 모델 위에서 이뤄진다(정본: 09):
  - 출처 커버리지 = `artifact`당 `citation` 비율.
  - 품질 = `hitl_review` 반려율·재작업률.
  - 비용·성능 = `pipeline_step`의 토큰·비용·지연 메트릭.
  - 격리·감사 = `tenant_id` 경계 + 감사 로그.
  - 표준·보존 = `package`/`xapi_statement`/보존 메타.

## [미정·가정]

- **수치 미정**: 모든 축의 정량 목표(정확도·SLA·처리량·단가·보존 기간)는 미정 — 파일럿 측정 후 확정.
- **(가정)** 전송/저장 암호화, 로그 마스킹, 적응형 사고·스트리밍, 프롬프트 캐싱·배치 비용 절감, 미디어 자동 자막·다국어 처리.
- **(가정)** provider 추상화로 LLM/미디어/벡터스토어 폴백·교체 가능, 기본 LLM `claude-opus-4-8`.
- **미정**: 강격리 범위, 폴백 provider 채택, 지원 표준 버전·언어, 규제 적용 범위, 백업/복구·삭제 SLA, 저작권 귀속·동의 모델.
- **참조**: 시스템·엔티티 정본은 [09-ARCHITECTURE.md](./09-ARCHITECTURE.md), 도메인 상세는 04~08.
