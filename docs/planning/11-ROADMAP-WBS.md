# 로드맵 · WBS — 맑은스튜디오 (LMS AI Studio)

> 목적: 기능 범위(03-FEATURES)와 전략(02-STRATEGY)을 **단계별 마일스톤**으로 묶고, 프로젝트 관리 허브(malgn-studio-mng)의
> **WBS 단계(stage) 시드**로 직접 쓸 수 있게 정리한다.
> 원칙: **속도 우선** — MVP는 "단일 소스 → 산출물 3종(요약·슬라이드·퀴즈) → 출처 → 검수 → 표준 발행"의 한 줄 가치 체인을 완성하고 4대 해자 씨앗을 모두 포함.

---

## 1. 단계(Phase) 개요

| Phase | 목표 | 핵심 산출 | 해자 |
|---|---|---|---|
| **P0 기반** | 멀티앱 골격·인증·프로젝트/RBAC·스튜디오 셸 | 로그인, 워크스페이스, 빈 생성 워크스페이스 | — |
| **P1 (MVP)** | 신뢰형 생성 가치체인 1회 완성 | 문서/PPT → RAG·출처 → 요약·슬라이드·퀴즈 → HITL 승인 → 자사 LMS+SCORM 1.2 발행 + 스킬 태깅 씨앗 | ①②③④ 씨앗 |
| **P2 확장** | 멀티모달·표준·개인화 본격화 | 웹·영상 인제스트, 커리큘럼·콘티·문제은행·오디오·영상(외부 API), 환각 가드·출처 뷰어, 스킬 택소노미, SCORM 2004·xAPI, 검수 코멘트, admin/lms 연동 | ①②③ 심화 |
| **P3 고도화** | 적응형·자동화·분석 | 자동 스킬 매핑·적응형 경로·생성형 변주, 멀티소스 병합, 다중 LMS 동기화, 학습 분석·스킬 갭 | ② 완성 |

> 휴넷 연내 고도화 대비 **P1을 최단 경로로** — 영상/오디오(외부 API)는 P2로 미뤄 출시 속도 확보(02-STRATEGY §6·§9).

---

## 2. 마일스톤

- **M1 (P0 완료)**: 스튜디오 앱 셸 + 로그인 + 프로젝트 생성 + 소스 업로드 UI(저장만).
- **M2 (P1 핵심)**: 문서/PPT 인제스트 → RAG 인덱싱 → **출처 인용 포함** 요약·슬라이드·퀴즈 생성.
- **M3 (P1 완성 = MVP 출시)**: HITL 승인 게이트 + 자사 LMS 네이티브 발행 + SCORM 1.2 export + 스킬 태깅(씨앗). → **데모/영업 가능한 신뢰형 빌더**.
- **M4 (P2-a)**: 웹·영상 인제스트 + 커리큘럼·콘티 + 미디어(영상·오디오) 1차 벤더 연동.
- **M5 (P2-b)**: 스킬 택소노미 + SCORM 2004·xAPI/LRS + 검수 코멘트·재생성 + admin/lms 운영.
- **M6 (P3)**: 적응형·생성형 변주 + 학습 분석·스킬 갭 리포트.

## 4. WBS 초기 작업(wbs_item) — 스타터 시드 (선택)

> 관리 허브를 빈 상태로 시작해 화면 CRUD로 채워도 되나, 아래를 단계별 대표 작업 시드로 제안(날짜는 ISO, owner는 §아래 에이전트팀 역할로 매핑).

| 단계 | 작업(예시) | 산출물 |
|---|---|---|
| 1 기반 | 멀티앱 레포 스캐폴드(studio/admin/api/lms) | 빌드 골격 |
| 1 기반 | 인증·세션·RBAC·프로젝트/워크스페이스 | 로그인·권한 |
| 2 인제스트 | 문서/PPT 파서 + 청킹·정규화 | 소스→청크 |
| 2 인제스트 | 임베딩·벡터스토어 인덱싱 + 출처 매핑 | RAG 검색 |
| 3 생성 | 생성 오케스트레이션 + 요약·슬라이드·퀴즈 | 산출물 3종 |

| 6 스킬 | 스킬 사전 + 산출물 스킬 태깅(씨앗) | 태깅 |

## 5. 진척 집계 규칙 (블루프린트 준수)

- 전체/단계 진척 = **단계 가중평균**(03-FEATURES 우선순위와 무관하게 stage.weight 기준).
- 작업 카운트·구분 롤업은 작업(wbs_item) 기준.
- QA(9단계, weight 0)는 표시만, 전체 진척에 영향 없음.
