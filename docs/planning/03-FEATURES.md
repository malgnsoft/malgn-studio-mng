# 기능 정의 · 범위 — 맑은스튜디오 (LMS AI Studio)

> 목적: 02-STRATEGY의 4대 해자·속도 전략을 실제 기능으로 분해하고, MVP/Phase 2/Later 범위를 확정한다.
> 우선순위 표기: P0(MVP 필수) / P1(Phase 2) / P2(Later). 추정은 "(가정)".

---

## 1. 목적

- 전략(02-STRATEGY)을 **빌드 가능한 기능 단위**로 분해한다.
- MVP는 "속도+차별화" 원칙에 따라 **최소화하되, 4대 해자의 씨앗을 반드시 포함**한다.
- 산출물 8종: 학습 영상 · 학습 오디오 · 슬라이드 · 커리큘럼 · 콘티(스토리보드) · 퀴즈 · 문제은행 · 학습 요약.

---

## 2. 핵심 결정 / 접근

- **결정 1 — MVP 산출물 축소.** 8종 중 **학습 요약·슬라이드·퀴즈**를 P0로(파싱·생성 부담 낮고 그라운딩 적용 쉬움). 영상·오디오는 외부 API 연동으로 P0 말미~P1.
- **결정 2 — 해자 씨앗 4종을 MVP에 모두 심는다.** 그라운딩(출처), 표준 발행(SCORM 1종), HITL(승인 게이트 1종), 스킬(태깅 씨앗) — 깊이는 얕아도 존재시킨다.
- **결정 3 — 영상/오디오는 Buy.** 어댑터로 추상화해 벤더 교체 가능하게(02-STRATEGY §7).

---

## 3. 사용자 · 역할

| 역할 | 주 사용 기능 영역 |
|---|---|

## 5. 범위 매트릭스 (MVP / Phase 2 / Later)

> 원칙: MVP는 **단일 소스 → 핵심 산출물 3종 → 출처 표기 → 검수 → 표준 발행**의 한 줄 가치 체인을 완성한다. 4대 해자 씨앗을 모두 포함.

| 영역 | MVP (Phase 1) | Phase 2 | Later |
|---|---|---|---|
| ⓐ 인제스트 | 문서·텍스트·PPT, 청킹 | 웹 링크, 영상(STT) | 멀티소스 자동 병합 |
| ⓑ 생성 | 요약·슬라이드·퀴즈, 오케스트레이션 | 커리큘럼·콘티·문제은행·오디오·영상 | 멀티 산출물 일괄 재생성 |
| ⓒ 그라운딩 | RAG 인덱싱, 출처 인용 | 환각 가드, 출처 뷰어 | 교차소스 근거 검증 |
| ⓓ 스킬·개인화 | 스킬 태깅(씨앗) | 스킬 택소노미 | 자동 매핑·적응형·생성형 변주 |
| ⓔ 발행·표준 | 자사 LMS 발행, SCORM 1.2 | SCORM 2004·xAPI, 버전 관리 | 다중 LMS 동기화 |
| ⓕ HITL | 승인 게이트 | 코멘트·재생성 요청 | 검수 큐·SLA |
| ⓖ 협업·운영 | 워크스페이스, RBAC | 버전/이력, admin 연동 | 동시 편집 |
| ⓗ 분석 | (없음) | 활동 로그 | 학습 분석·스킬 갭 |

**MVP에 심긴 4대 해자 씨앗(확인용):**

- 해자 ① 발행+표준 → 자사 LMS 발행 + SCORM 1.2 (P0)
- 해자 ② 스킬 → 스킬 태깅 씨앗 (P0)
- 해자 ③ 그라운딩 → RAG 인덱싱 + 출처 인용 (P0)
- 해자 ④ HITL → 승인 게이트 (P0)

## 6. 미정 · 가정

- (가정) MVP 핵심 산출물 3종(요약·슬라이드·퀴즈) 선정은 파싱·그라운딩 난이도 기준 추정 → 사용자 수요로 재검증.
- (가정) 영상·오디오를 P1로 미룬 것은 외부 API 연동 일정 의존 → 벤더 확정 시 P0 승격 가능.
- (가정) xAPI는 LRS 연동 전제이며 보유 여부 미확인.
- (미정) 스킬 사전(태깅 씨앗용) 초기 출처 — 기존 LXP 스킬 프로파일 재사용 가능 여부.
- (미정) 인제스트 파서 Build/Buy 경계(문서·PPT 추출 라이브러리 선택).
- (미정) RAG 스택(임베딩·벡터스토어) 및 LLM 벤더.
